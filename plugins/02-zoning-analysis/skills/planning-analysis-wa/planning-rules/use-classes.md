# Use Classes — P / D / A / X permissibility

Every LPS in WA has a Table of Uses (also called the Zoning Table) that lists **use classes** down the rows and **zones** across the columns. Each cell shows how the use is treated in that zone.

## The current four-code system

From 2015 (with the deemed provisions), WA uses a four-code system:

| Code | Name | Meaning | Advertising |
|---|---|---|---|
| **P** | Permitted | Use is permitted — no DA required if it also meets all deemed provisions and R-Code DTS. For minor works, exemption from DA may apply | Not required |
| **D** | Discretionary | Use may be approved at the discretion of the Responsible Authority | Usually at discretion |
| **A** | Advertised discretionary | As `D`, plus mandatory public advertising (usually 14+ days) before determination | Required |
| **X** | Not permitted | Use is prohibited in the zone. A DA for this use cannot be approved under the current LPS | — |

A blank cell is treated as `X` unless the LPS text says otherwise.

## Older codes you may encounter in legacy LPSs

Some older LPSs (pre-2015) used a different code set. If you find any of these, translate as follows:

| Legacy code | Treat as | Notes |
|---|---|---|
| P | P | Usually unchanged |
| AA | D | "Additional Approval" required — discretionary |
| SA | A | "Special Approval" — discretionary + advertising |
| IP | P | "Incidental to Permitted" — treat as P if incidental to a P use |
| X | X | Prohibited |

Cite the original code in brackets if using the legacy system, e.g. `"D (formerly AA per LPS [number] cl. [x])"`.

## How to check permissibility

1. Identify the **LPS zone** for the site (from LGA zoning map)
2. Identify the **use class** that best matches the proposed use (from the LPS use class definitions or Sch. 2 of the Regulations)
3. Look up the cell in the LPS Table of Uses
4. If ambiguous — e.g. the proposed use could be either Multiple Dwelling or Mixed Use — note the interpretation and cite the LPS definition you're relying on

If the use is not clearly defined:
- Check the **Regulations Sch. 2 cl. 1.8** for standard definitions (these apply to every LPS unless expressly varied)
- Check the LPS **glossary / Schedule of Definitions** for LGA-specific definitions
- If still ambiguous, the Responsible Authority may issue a **use interpretation** under Sch. 2 cl. 65

## Common residential use classes

| Use class | Definition (summary) | Typical R-Code pathway |
|---|---|---|
| **Single House** | A single dwelling with its own land title | Vol. 1 |
| **Ancillary Dwelling** | A second, smaller, subordinate dwelling on the same lot (formerly "granny flat") | Vol. 1 |
| **Grouped Dwelling** | Dwellings sharing a lot (strata-titled) with separate entries; low-rise, side-by-side or attached | Vol. 1 |
| **Multiple Dwelling** | Apartments — dwellings in a multi-storey building sharing circulation | Vol. 2 (if R30+ and 4+ storeys) |
| **Mixed Use** | Combination of residential + commercial on the same lot | Vol. 2 + LPS commercial controls |
| **Short-Term Dwelling** | Short-stay accommodation (< 3 months) | Usually a separate use class — check LPS |

## Practical notes

- **Use the LPS's own definitions first**, then fall back to the Regulations Sch. 2
- If the proposed use doesn't fit any listed class, it may be `Use not listed` — Sch. 2 cl. 4.4 gives the Responsible Authority discretion to determine these as `D` or `A`
- Many LPSs have **Additional Use Areas** — small spatial overlays that permit or restrict specific uses on specific lots (listed in the LPS as a table, not on the zoning map). Check the LPS text for Additional Use Areas affecting the site
- **Restricted Uses** — similar to Additional Uses but restrictive — listed in the LPS
- Even if a use is `P`, the proposal still needs to meet all bulk controls (plot ratio, height, setbacks), all R-Code DTS or design principles, and all LPP requirements
